# 🎯 AI Agent Mission: Event & Market System Consolidation

আপনার কাজ হলো Plokymarket-এর Event এবং Market creation প্রসেসকে একটি প্রোডাকশন-রেডি **Service Layer Architecture**-এ রূপান্তর করা। এই মুহূর্তে লজিকগুলো Scattered অবস্থায় আছে, যা রিফ্যাক্টর করে "Instant Go-Live" নিশ্চিত করতে হবে।

## 1. Core Architecture Rules (ভুল করবেন না)
- **Modular Isolation:** শুধুমাত্র নির্দিষ্ট মডিউলে কাজ করবেন।
- **Interface First:** কোড লেখার আগে `src/types/market-system.ts` চেক বা আপডেট করবেন।
- **No Deletion:** কোনো এক্সিস্টিং API রুট ডিলিট করবেন না, শুধু লজিক বের করে সার্ভিসে আনবেন।
- **Transaction Safety:** ডাটাবেস অপারেশনগুলো `supabase.rpc` বা ট্রানজেকশনে রাখবেন।

## 2. Entity Relationship Foundation
- **Event (Container):** বাস্তব জীবনের ঘটনা (যেমন: বিপিএল ২০২৬)। টেবিল: `events`.
- **Market (Tradable):** YES/NO প্রশ্ন (যেমন: রংপুর কি জিতবে?)। টেবিল: `markets`.
- **Relationship:** 1 Event -> N Markets.

## 3. Tech Stack Requirements
- **Backend Service:** `src/lib/services/MarketService.ts` এবং `EventService.ts`.
- **Realtime:** Supabase Realtime (Walrus setup).
- **Matching Engine:** `orders` টেবিলে Initial Liquidity সিডিং।

## 4. Step-by-Step Implementation Plan

### Phase 1: Service Layer Extraction (Backend)
1. **EventService:** `app/api/admin/events/create/route.ts` থেকে লজিক বের করে `lib/services/EventService.ts` এ মাইগ্রেট করুন।
2. **MarketService:** `app/api/admin/market-creation/deploy/route.ts` থেকে লজিক বের করে `lib/services/MarketService.ts` এ আনুন।
3. **Liquidity Seeding:** `MarketService` এর ভেতরে `seedInitialOrderbook(marketId, amount)` মেথড যোগ করুন যা সরাসরি `orders` টেবিলে YES এবং NO লিমিট অর্ডার প্লেস করবে।

### Phase 2: Instant Go-Live Mechanism
- মার্কেট স্ট্যাটাস `pending` থেকে `active` করার সাথে সাথে লিকুইডিটি বট রান করতে হবে।
- এজেন্টের কাজ: মার্কেট তৈরি হওয়ার পরপরই যাতে ইউজারের কাছে Buy বাটন কাজ করে, তার জন্য অন্তত ১০০টি করে শেয়ার ৫% স্প্রেডে অর্ডারে বসানো নিশ্চিত করা।

### Phase 3: Frontend Sync (Admin & User)
1. **Admin Wizard:** `components/admin/MarketCreationWizard.tsx` ফাইলটি আপডেট করুন যাতে এটি নতুন `MarketService` ব্যবহার করে।
2. **Realtime Hook:** `hooks/useMarketsRealtime.ts` তৈরি বা আপডেট করুন যাতে নতুন মার্কেট তৈরির সাথে সাথে ইউজার ড্যাশবোর্ডে তা রিফ্রেশ ছাড়াই দেখা যায়।

## 5. Underlying Issues to Fix (Crucial)
- **Field Mismatch:** এডমিন প্যানেলে 'ends_at' ফিল্ডটি যেন `markets.trading_closes_at` এবং `events.ends_at` উভয়ের সাথে সিঙ্ক থাকে।
- **SQL RLS:** নিশ্চিত করুন যে `service_role` দিয়ে এডমিন কাজ করলেও ইউজারের ক্ষেত্রে যেন `anon` কি কাজ করে।
- **Hardcoded Keys:** সব এপিআই এন্ডপয়েন্ট থেকে হার্ডকোডেড কী সরিয়ে `process.env` ব্যবহার করুন।
